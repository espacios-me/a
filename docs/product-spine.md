# Product spine

Atom is intentionally centered on:

1. Sign-in / SSO entry
2. Integrations control center
3. Messaging workspace

## Intentional unfinished areas

- Production OAuth/SSO token exchange and provider callbacks
- Persistent storage (D1/KV) replacing in-memory session state
- Live provider API execution inside assistant/tool calls

These are deliberately represented as preview-ready architecture, not fake production completion.
